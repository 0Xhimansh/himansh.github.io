// 05_PROJECT_DEEP_DIVES

CASE STUDY ID: PROJ_811 CRITICAL

Enterprise SaaS Authorization Chain Break

Problem: A multi-tenant SaaS platform trusted client-controlled identifiers across chained API calls, allowing lateral movement between tenant boundaries.

Surface: GraphQL + REST APIs

Vuln Class: BOLA / Authorization Logic Flaw

Impact: Cross-tenant data exposure across enterprise customers.

Fix: Centralized authorization layer enforcing tenant-bound object ownership on every resolver.

CASE STUDY ID: PROJ_904 HIGH

Active Directory Trust Escalation

Problem: Misconfigured inter-domain trust combined with weak ACLs enabled privilege escalation from a child domain into forest-level admin.

Surface: Active Directory / Kerberos

Vuln Class: Trust Abuse / ACL Misconfiguration

Impact: Full forest compromise from low-privileged domain user.

Fix: Hardened trust relationships, DACL auditing, and tiered admin model.
